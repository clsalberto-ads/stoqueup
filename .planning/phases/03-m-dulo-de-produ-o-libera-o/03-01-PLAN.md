<task id="2" type="execute">
<read_first>
- src/lib/inventory-actions.ts
</read_first>
<action>
Adicionar a Server Action `completeProduction` ao arquivo `src/lib/inventory-actions.ts`.
A action deve:
1. Receber `taskId`, `productId` e `actualQuantity`.
2. Verificar autenticação.
3. Iniciar uma transação `db.transaction`.
4. Atualizar o status da tarefa para `COMPLETED`.
5. Incrementar o estoque do produto (`currentStock = currentStock + actualQuantity`).
6. Trava de Segurança: Se `newStock > qtdMaxima`, truncar o valor para `qtdMaxima` ou lançar erro (conforme decidido em D-05). Vamos truncar para o máximo permitido e logar o ajuste.
7. Inserir log em `inventoryLogs` com tipo `PRODUCTION`.
8. Retornar sucesso.
</action>
<acceptance_criteria>
- `completeProduction` implementada com transação.
- Estoque é incrementado corretamente.
- Travas de `qtdMaxima` são respeitadas.
</acceptance_criteria>
</task>
